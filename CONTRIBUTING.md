# Contributing

## Guidelines

- Keep changes focused and reviewable.
- Do not commit real secrets or personal data.
- Preserve existing behavior unless a change is explicitly scoped.
- Update multilingual documentation when behavior changes.
- Add reviewed screenshots only when they contain no sensitive data.

## Pull Request Checklist

- [ ] No business logic changed unexpectedly.
- [ ] No secret or real .env value was added.
- [ ] Documentation matches the code.
- [ ] Relevant validation was run.
